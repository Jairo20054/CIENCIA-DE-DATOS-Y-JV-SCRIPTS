1. Método para no saturarte

El objetivo no es consumir muchos temas: es dominar un concepto por clase y convertirlo en práctica.

1

Entender.
Pide una explicación sencilla, ejemplos y analogías.

2

Implementar.
Escribe código propio, aunque sea pequeño.

3

Romper.
Cambia datos, provoca errores y aprende a depurar.

4

Explicar.
Resume el concepto como si fueras a enseñarlo.

No pases a la siguiente clase si solo viste videos o leíste teoría. Debes poder responder: ¿qué es?, ¿para qué sirve?, ¿cómo se implementa?, ¿qué errores comunes tiene?

Esta pestaña es tu glosario profesional. Úsala cuando escuches una palabra técnica y no quieras perderte. Puedes buscar por término, filtrar por categoría y copiar un prompt para pedirle a ChatGPT una explicación completa.

Forma correcta de estudiar esta pestaña: no memorices todo de una vez. Cada semana escoge 10 términos relacionados con la clase actual, pídele ejemplos a ChatGPT y crea una frase propia para explicar cada término.

5. Prompts para estudiar cada clase

Usa estos prompts con el concepto de la clase. Cambia lo que está entre corchetes.

Prompt maestro

Actúa como profesor universitario y mentor senior de Ingeniería de Software. Enséñame [CONCEPTO DE LA CLASE] desde cero, sin saturarme. Quiero: 1. Explicación sencilla. 2. Para qué sirve en software real. 3. Ejemplo en JavaScript/TypeScript. 4. Errores comunes. 5. 5 ejercicios progresivos. 6. Un mini reto aplicado. 7. Preguntas para verificar si entendí.

Prompt para ejercicios

Dame ejercicios sobre [CONCEPTO DE LA CLASE]. No me des la solución al inicio. Organízalos en: - 3 fáciles - 3 medios - 2 aplicados a un proyecto real Cuando te envíe mi código, evalúa claridad, errores, casos borde y mejora profesional.

Prompt para proyecto

Quiero aplicar [CONCEPTO DE LA CLASE] en un mini proyecto. Dame: - requisitos - pasos pequeños - estructura de archivos - criterios de aceptación - errores comunes - cómo subirlo a GitHub Guíame paso a paso, no me des todo resuelto de una vez.

Prompt para evaluación

Evalúame sobre [CONCEPTO DE LA CLASE] como si fuera una entrevista técnica. Hazme preguntas de menor a mayor dificultad. Después de cada respuesta, califícame de 1 a 10, corrige mi explicación y dime qué debo practicar antes de avanzar.

8. Evaluación para avanzar de clase

Antes de marcar una clase como terminada, revisa estos criterios.

Comprensión

  • Puedo explicar el concepto sin leer.
  • Sé cuándo usarlo y cuándo no.
  • Conozco mínimo 3 errores comunes.

Práctica

  • Hice ejercicios propios.
  • Subí código a GitHub.
  • Probé casos normales y casos borde.

Profesionalismo

  • El código tiene nombres claros.
  • El README o bitácora está actualizado.
  • Puedo defender mi solución en entrevista.
Tu meta no es “terminar rápido”. Tu meta es que cada semana suba tu nivel técnico, tu criterio y tu portafolio.
