3. Ruta de estudio de 12 meses

Cada mes tiene conceptos, práctica, entregable y prompts para estudiar con ChatGPT. Marca lo que completes; el avance queda guardado en este navegador.

01Mes

Fundamentos de programación y pensamiento computacional

Objetivo: escribir programas pequeños sin copiar, entendiendo datos, flujo y errores.

Conceptos: variables, tipos de datos, operadores, condicionales, ciclos, funciones, arrays, objetos, módulos, entrada/salida, debugging, Git básico.

Proyecto: calculadora de presupuesto personal + conversor de unidades + menú por consola.

02Mes

JavaScript profundo + programación orientada a objetos

Objetivo: dominar objetos, clases, módulos, asincronía y estructura de código.

Conceptos: scope, closures, this, prototypes, clases, encapsulación, herencia, composición, módulos ES, async/await, promesas, manejo de errores.

Proyecto: gestor de tareas con usuarios, categorías, filtros y persistencia local.

03Mes

Estructuras de datos y algoritmos

Objetivo: resolver problemas pensando en eficiencia, no solo en que funcione.

Conceptos: Big O, arrays, listas, pilas, colas, hash maps, sets, árboles, grafos, recursión, búsqueda, ordenamiento, two pointers, sliding window.

Proyecto: biblioteca de estructuras de datos + 60 problemas resueltos y explicados.

04Mes

Bases de datos, SQL y modelado

Objetivo: diseñar datos correctamente y consultar información con criterio.

Conceptos: modelo entidad-relación, normalización, claves, relaciones, SQL, joins, índices, transacciones, vistas, triggers básicos, PostgreSQL, seguridad de datos.

Proyecto: base de datos para propiedades/arriendos con usuarios, favoritos, reservas y pagos simulados.

05Mes

Backend profesional con Node.js, APIs y autenticación

Objetivo: construir servicios seguros, mantenibles y conectados a base de datos.

Conceptos: HTTP, REST, Express/NestJS, controladores, servicios, repositorios, validación, JWT, cookies, hashing, middlewares, manejo global de errores, paginación, filtros.

Proyecto: API de marketplace de propiedades con auth, CRUD, filtros, roles y documentación.

06Mes

Frontend moderno con React y TypeScript

Objetivo: crear interfaces limpias, rápidas, responsivas y conectadas a una API.

Conceptos: componentes, props, estado, hooks, contexto, rutas, formularios, validación, consumo API, loading/error states, accesibilidad, responsive, diseño de componentes.

Proyecto: frontend de propiedades con búsqueda, filtros, detalle, login y panel básico.

07Mes

Ingeniería de software, testing y calidad

Objetivo: pasar de “programar” a construir software sostenible.

Conceptos: requisitos, historias de usuario, casos de uso, UML básico, pruebas unitarias, integración, E2E, TDD básico, mocks, cobertura, refactor, code review, deuda técnica.

Proyecto: suite de pruebas para backend y frontend + tablero de issues y documentación técnica.

08Mes

Arquitectura de software y patrones

Objetivo: diseñar sistemas que crezcan sin volverse desordenados.

Conceptos: SOLID, clean architecture, capas, MVC, hexagonal, patrones creacionales/estructurales/comportamiento, DTO, repositorios, servicios, eventos, documentación C4.

Proyecto: rediseñar la API en arquitectura limpia + diagrama C4 + decisiones arquitectónicas ADR.

09Mes

Seguridad, redes y fundamentos de infraestructura

Objetivo: entender cómo viaja la información y cómo proteger una app web.

Conceptos: DNS, TCP/IP, HTTP/HTTPS, CORS, cookies seguras, OWASP Top 10, XSS, CSRF, SQL injection, rate limit, logging, backups, variables de entorno, permisos.

Proyecto: auditoría de seguridad de tu app + checklist OWASP + mejoras reales.

10Mes

Sistemas operativos, Docker, despliegue y CI/CD

Objetivo: ejecutar, desplegar y monitorear software como en un entorno profesional.

Conceptos: procesos, memoria, concurrencia, archivos, Linux, terminal, Docker, contenedores, variables de entorno, GitHub Actions, despliegue, monitoreo, logs.

Proyecto: dockerizar frontend/backend/base de datos + pipeline CI/CD + despliegue en la nube.

11Mes

Datos, inteligencia artificial y aprendizaje de máquina aplicado

Objetivo: usar datos para construir funciones inteligentes sin perder fundamentos.

Conceptos: Python básico para datos, pandas, limpieza de datos, regresión, clasificación, métricas, embeddings, recomendadores, APIs de IA, límites éticos, privacidad.

Proyecto: recomendador simple de propiedades + dashboard de datos + explicación de métricas.

12Mes

Proyecto final: sistema completo de nivel portafolio

Objetivo: demostrar que puedes diseñar, construir, probar, desplegar y explicar un producto real.

Conceptos: arquitectura, backlog, releases, métricas, documentación, observabilidad, rendimiento, seguridad, experiencia de usuario, presentación técnica.

Proyecto: plataforma full-stack publicada con README profesional, video demo, diagramas, pruebas, seguridad y mejoras pendientes.

6. Prompts profesionales para estudiar con ChatGPT

Copia, cambia el tema entre corchetes y úsalo. No pidas solo explicación: pide práctica, errores, evaluación y proyecto.

Prompt maestro para aprender un concepto

Actúa como profesor universitario de Ingeniería de Software. Enséñame [CONCEPTO] desde cero hasta nivel intermedio. Incluye: 1. Definición sencilla. 2. Para qué sirve en software real. 3. Ejemplo en JavaScript/TypeScript. 4. Errores comunes. 5. 10 ejercicios progresivos. 6. Un mini proyecto. 7. Preguntas tipo examen. 8. Revisión de mi solución cuando te la envíe.

Prompt para practicar algoritmos

Quiero practicar [TEMA DE ALGORITMOS]. Dame 5 problemas: fácil, medio, medio+, difícil y reto. No me des la solución al inicio. Primero dame pistas por niveles. Cuando te envíe mi código, evalúa: - correctitud - complejidad Big O - legibilidad - casos borde - cómo mejorarlo.

Prompt para proyectos reales

Ayúdame a construir una funcionalidad real para mi portafolio: [DESCRIBE LA FUNCIÓN]. Quiero que lo hagas como senior: - requisitos - modelo de datos - endpoints - componentes frontend - validaciones - casos de prueba - riesgos de seguridad - criterios de aceptación No escribas todo por mí; guíame paso a paso y revisa mi avance.

Prompt para entender errores

Tengo este error: [PEGA EL ERROR] Y este código: [PEGA EL CÓDIGO] Explícame: 1. Qué significa el error en palabras sencillas. 2. Dónde está la causa probable. 3. Cómo depurarlo paso a paso. 4. Solución mínima. 5. Solución profesional. 6. Cómo evitar que vuelva a pasar.

Prompt para code review

Actúa como revisor senior de código. Revisa este código: [PEGA EL CÓDIGO] Evalúa: - claridad - duplicación - nombres - seguridad - rendimiento - arquitectura - pruebas necesarias Dame una versión mejorada y explícame cada cambio.

Prompt para explicar como ingeniero

Evalúame como si estuviera en una entrevista técnica. Pregúntame sobre [CONCEPTO]. Hazme preguntas de menor a mayor dificultad. Después de cada respuesta mía, califícame de 1 a 10 y dime qué debo mejorar. No avances hasta que mi explicación sea clara.

8. Libros recomendados por etapa

No necesitas leerlos todos de una vez. Usa los libros como apoyo mientras construyes proyectos.

Etapa Libro Para qué usarlo Cómo estudiarlo
Programación base Eloquent JavaScript · Marijn Haverbeke Aprender JavaScript, funciones, estructuras, objetos y programación web. Leer capítulos con ejercicios y pasar ejemplos a tu propio repositorio.
JavaScript profundo You Don’t Know JS Yet · Kyle Simpson Entender scope, closures, this, tipos y mecanismos internos. Usarlo después del mes 1; no memorizar, experimentar en código.
Algoritmos iniciales Grokking Algorithms · Aditya Bhargava Comprender algoritmos con explicaciones visuales y problemas prácticos. Resolver cada algoritmo en JavaScript y Python.
Algoritmos universitarios Introduction to Algorithms · Cormen, Leiserson, Rivest, Stein Base formal para análisis de algoritmos y estructuras avanzadas. Consultar por temas; no leer lineal completo al inicio.
Código profesional Clean Code · Robert C. Martin Nombres, funciones, clases, errores, limpieza y mantenibilidad. Aplicar capítulo por capítulo a tus propios proyectos.
Oficio del programador The Pragmatic Programmer · Hunt & Thomas Hábitos, criterio, responsabilidad, comunicación y carrera técnica. Leer una sección semanal y convertirla en regla personal.
Refactorización Refactoring · Martin Fowler Mejorar diseño sin cambiar comportamiento externo. Usarlo en el mes 7 para mejorar código ya construido.
Patrones Head First Design Patterns · Freeman & Robson Aprender patrones con ejemplos orientados a objetos. Aplicar solo patrones que resuelvan problemas reales de tu app.
Arquitectura Clean Architecture · Robert C. Martin Separación de responsabilidades, capas y reglas de dependencia. Leer junto con un refactor de tu backend.
Bases de datos Database System Concepts · Silberschatz, Korth, Sudarshan Modelo relacional, consultas, transacciones, índices y teoría. Consultar mientras diseñas PostgreSQL.
Redes Computer Networking: A Top-Down Approach · Kurose & Ross Entender HTTP, DNS, TCP/IP y servicios de red. Leer capítulos de web, transporte y seguridad.
Sistemas operativos Operating Systems: Three Easy Pieces · Arpaci-Dusseau Procesos, memoria, concurrencia y persistencia. Complementar con prácticas en Linux y Docker.
Sistemas de datos Designing Data-Intensive Applications · Martin Kleppmann Datos, consistencia, replicación, particionamiento y sistemas distribuidos. Leer después de bases de datos y backend.
Ingeniería real Software Engineering at Google · Winters, Manshreck, Wright Escala, prácticas, cultura técnica, mantenibilidad y trabajo en equipo. Usarlo para madurar como ingeniero, no solo como programador.
Seguridad Secure by Design · Johnsson, Deogun, Sawano Diseñar software con seguridad desde el modelo y el código. Aplicarlo en autenticación, permisos y validación.
Machine Learning Hands-On Machine Learning · Aurélien Géron Aprender ML con ejemplos prácticos en Python. Usarlo en el mes 11 con un proyecto pequeño.

9. Rúbrica para saber si vas bien

Cada mes evalúate. No basta con ver videos: tienes que construir, explicar y sostener tu código.

Nivel básico

Puedo seguir tutoriales y hacer ejercicios aislados.

  • Entiendo sintaxis.
  • Hago programas pequeños.
  • Uso Git básico.

Nivel intermedio

Puedo construir funcionalidades propias con estructura y criterio.

  • Diseño modelos de datos.
  • Creo APIs y frontend conectado.
  • Escribo pruebas y documentación.

Nivel profesional

Puedo defender decisiones técnicas y mantener software real.

  • Diseño arquitectura escalable.
  • Pienso en seguridad, rendimiento y mantenimiento.
  • Despliego, monitoreo y explico el sistema.
Examen mensual sugerido
  1. Explica el tema del mes sin mirar apuntes.
  2. Resuelve un problema nuevo en máximo 45 minutos.
  3. Agrega una funcionalidad al proyecto sin romper lo anterior.
  4. Escribe al menos 3 pruebas.
  5. Haz un commit claro y actualiza README.
  6. Pide a ChatGPT que te entreviste sobre el tema y califica tus respuestas.
